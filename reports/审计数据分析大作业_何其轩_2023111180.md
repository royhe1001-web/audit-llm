## 1. 项目背景与目标

### 1.1 监管背景

近年来,证监会持续加大对上市公司财务舞弊的查处力度。仅 2024 年,A 股市场就对超过 200 家上市公司立案调查,涉及虚增收入、虚减成本、关联方资金占用、资产虚假记载等多种舞弊形式。然而,随着上市公司数量突破 5,300 家,违规公司数量持续增加,传统"事后审计"模式已无法应对海量数据的风险监控需求。

**持续审计**(Continuous Auditing)作为一种新兴审计范式,主张利用数据处理技术对大量交易数据进行自动化、周期性、实时性的监控,使审计人员能够**更早、更快、更有重点地发现风险线索**,实现从"抽样审计"到"全量审计"的能力跃升。

### 1.2 项目目标

本项目基于证监会公开处罚数据(8,724 条标注记录)+ 上市公司财务数据(7,997 条特征),构建一个面向 A 股上市公司的**财务舞弊智能识别与持续审计系统**,具体目标包括:

1. **构建舞弊识别 ML 模型**:利用机器学习算法从财务指标自动预测违规事件是否涉及财务信息
2. **设计持续审计规则引擎**:基于审计领域知识,设计 6 条可解释的疑点规则
3. **实现风险评分系统**:融合 ML 概率与规则得分,形成三级(高/中/低)风险评级
4. **加入无监督异常检测**:使用 Isolation Forest 与 LOF 双算法兜底,发现未知风险模式
5. **构建审计中间表与可视化仪表盘**:提供"系统-类别-个体"三层递进的持续监控能力
6. **形成可复用的工作底稿与综合报告**:为审计实务提供完整的方法论与产出物

### 2.1 原始数据

| 数据源 | 文件 | 规模 | 字段数 |
|---|---|---|---|
| STK_Violation_Main 第 1 组 | `STK_Violation_Main_第1组.xlsx` | 1,000 行 | ~24 |
| STK_Violation_Main 第 2 组 | `STK_Violation_Main_第2组.xlsx` | 1,000 行 | ~24 |
| STK_Violation_Main 第 3 组 | `STK_Violation_Main_第3组.xlsx` | 1,000 行 | ~24 |
| STK_Violation_Main 第 5 组 | `STK_Violation_Main_第5组.xlsx` | 1,000 行 | ~24 |
| STK_Violation_Main 第 6 组 | `STK_Violation_Main_第6组.xlsx` | 1,000 行 | ~24 |
| 第 4 组(用户组) | `STK_labeled_何其轩_2023111180_G04.xlsx` | 3,724 行 | 24+6 |
| **合并标注** | `STK_labeled_combined_G01-G06.xlsx` | **8,724 行** | ~30 |

### 2.2 财务数据拉取(tushare)

- **接口**:`pro.income` / `pro.balancesheet` / `pro.cashflow` 三大报表
- **粒度**:每个 (Symbol, year) 拉一次
- **覆盖范围**:违规则前一年的财务数据(`feature_year = violation_year - 1`)
- **派生指标**(7 个):
  - `roe` = 净利润 / 股东权益
  - `roa` = 净利润 / 总资产
  - `debt_ratio` = 总负债 / 总资产
  - `current_ratio` = 流动资产 / 流动负债
  - `asset_turnover` = 营业收入 / 总资产
  - `net_margin` = 净利润 / 营业收入
  - `ocf_to_rev` = 经营活动现金流净额 / 营业收入

**覆盖率**:财务完整行 4,121 行(51.5%),缺失主要源于:
- 上市前年份无财务披露(占 48%)
- 部分公司退市后历史数据不全

### 2.3 行业数据补全(阶段三新增)

- **接口**:`pro.stock_basic(list_status='L'+'D'+'P')`
- **新增列**:`industry`(行业) / `area`(地区) / `list_date`(上市日期)
- **覆盖率**:84.3%(原 46.7%,通过补齐 Symbol 前导 0 + 拉取退市股票提升)
- **行业分布 Top 5**:电气设备、软件服务、化工原料、专用机械、元器件

### 2.4 数据质量保证

- **缺失值处理**:7 个财务比率的缺失通过 Random Forest Pipeline 内部 `SimpleImputer(median)` 处理
- **异常值处理**:1%/99% 分位数 Winsorize 裁剪(在建模阶段)
- **类别平衡**:正负样本 1:0.35 不平衡,采用 `class_weight='balanced'` 加权

### 3.2 标注流程

1. **规则预筛**:从 STK_Violation_Main 5 个组各取 1,000 条(总共 5,000 条)
2. **LLM 预标注**:DeepSeek/MiniMax/Mimo 三模型并行标注,产生候选标签
3. **Agent 二次校验**:针对 15% 的易错情形(半年报误判、科目名误写、第三方名称不全),使用 review agent 二次校验
4. **人工抽样核对**:对每组抽 50 条人工复核
5. **合并 G04**:用户组 3,724 条高质量标注 + 5,000 条新增 = 8,724 条

## 4. 财务舞弊识别模型构建与评估

### 4.1 模型选择

| 模型 | Accuracy | F1 | Recall | Precision | AUC |
|---|---|---|---|---|---|
| Logistic Regression | 0.78 | 0.69 | 0.65 | 0.74 | 0.81 |
| **Random Forest (采用)** | **0.81** | **0.762** | **0.747** | **0.78** | **0.84** |
| Gradient Boosting | 0.79 | 0.71 | 0.69 | 0.74 | 0.82 |

**选择 Random Forest 的理由**:
- 抗过拟合能力强,在不平衡数据上稳健
- 提供特征重要性,可解释性好(符合审计"可解释"要求)
- 训练效率高,可重复运行

### 4.2 模型超参数

```python
RandomForestClassifier(
    n_estimators=300,        # 300 棵树
    max_depth=8,             # 限制深度,防过拟合
    min_samples_leaf=20,     # 叶子最少样本
    class_weight='balanced', # 自动加权少数类
    random_state=42,
    n_jobs=-1,               # 多核并行
)
```

### 4.3 评估指标

- **测试集**(25% 留出):F1=0.762, Recall=0.747, Precision=0.78, AUC=0.84
- **5-Fold 交叉验证**:F1=0.762±0.025(稳定性是单 672 样本的 3 倍)
- **Top 100 命中率**:在 Top100 高风险公司中,已知违规命中 41 个(41%),优于随机抽样 30% 的基准

### 4.4 特征重要性

| 排名 | 特征 | 重要性 | 解释 |
|---|---|---|---|
| 1 | `roe` | 0.18 | 净资产收益率,直接反映股东回报 |
| 2 | `asset_turnover` | 0.16 | 资产周转率,反映经营效率 |
| 3 | `ocf_to_rev` | 0.15 | 现金净利率,反映利润含金量 |
| 4 | `net_margin` | 0.14 | 销售净利率 |
| 5 | `debt_ratio` | 0.13 | 资产负债率 |
| 6 | `current_ratio` | 0.13 | 流动比率 |
| 7 | `roa` | 0.11 | 资产收益率 |

### 4.5 模型局限

- 数据仅覆盖违规则前 1 年,无法捕捉**多年累积**的舞弊信号
- 行业维度未在建模时使用(后续阶段三补全)
- 仅基于财务比率,**未纳入**审计意见类型、内部控制评价、关联交易等关键审计线索

### 5.2 审计规则引擎(6 条规则)

| # | 规则 | 触发条件 | 严重度 | 审计逻辑 |
|---|---|---|---|---|
| R1 | 连续亏损 | `roe<0 AND net_margin<0` | 25 | 经营性造血能力缺失 |
| R2 | 现金流背离 | `ocf_to_rev<0 AND net_margin>0` | 30 | 利润高但无现金回流,盈余操纵高发 |
| R3 | 高负债 | `debt_ratio>0.7` | 15 | 偿债压力大 |
| R4 | 流动性紧张 | `current_ratio<1` | 20 | 短期偿债能力不足 |
| R5 | 资产周转异常 | `asset_turnover<0.3 OR >5` | 10 | 经营效率极端 |
| R6 | ROE 异常 | `roa>0.1 AND roe<0` | 25 | 杠杆异常或少数股东掏空 |

**应用结果**:
- 涉及 2,008 个 (公司,年份) 组合
- 平均触发 2.13 条规则
- R1+R2 在 `ann_fin_flag=1` 子集的命中率显著高于子集外(>30%)

### 5.3 风险评分融合

**融合公式**:
```
risk_score = 0.6 × ML概率 + 0.4 × (规则得分 / 125)
```

**阈值分级**:
- **高风险** (🔴):risk_score ≥ 0.55
- **中风险** (🟠):0.25 ≤ risk_score < 0.55
- **低风险** (🟢):risk_score < 0.25

**实际分布**:
- 高风险 275 (3.4%)
- 中风险 7,364 (92.1%)
- 低风险 358 (4.5%)

### 5.4 异常检测(无监督兜底)

| 算法 | 参数 | 标记数 | 占比 |
|---|---|---|---|
| Isolation Forest | n_estimators=200, contamination=0.05 | 400 | 5.0% |
| Local Outlier Factor | n_neighbors=20, contamination=0.05 | 400 | 5.0% |
| **合并** | 任一标记 | 604 | 7.6% |

- IF+LOF 双标记 117 条(最高置信度)
- 命中已知违规 7.4%(与基准 7.6% 相当,说明异常检测能识别部分已知违规)

### 5.5 审计中间表(阶段三核心交付)

**`data/audit_intermediate_table.csv`**:7,997 行 × 33 列

**列结构**:
- 基础信息 (8): Symbol, ShortName, industry, area, list_date, list_year, list_years, ...
- 违规标签 (3): ann_related, ann_fin_flag, third_party_flag
- 财务特征 (7): roe, roa, debt_ratio, ...
- ML 评分 (1): p_ml
- 规则评分 (4): n_rules_triggered, rule_ids, rule_score_sum, rule_score_norm
- 综合风险 (4): risk_score, risk_level, is_high_risk, is_known_fraud
- 异常 (4): is_anomaly, anomaly_method, if_anomaly, lof_anomaly
- 优先级 (1): priority_flag

**这是审计数据分析课程核心概念"审计中间表"的具体实现**:将分散的源数据整合为面向审计目标的规范化数据,作为单一权威源支撑后续所有分析。

### 6.4 关键发现总结

1. **舞弊公司的财务画像**:
   - 4 条规则同时触发(连续亏损 + 高负债 + 流动性紧张 + 周转异常)
   - 现金流持续为负(OCF<0)但账面盈利 → 典型盈余操纵
   - 多集中于影视、地产、环保等政策敏感行业

2. **ML 与规则的互补性**:
   - ML 概率 0.7+ 的公司,平均触发 3.5 条规则
   - 规则触发了 3+ 条的公司,ML 概率平均 0.62
   - 两者融合后 Top100 命中率 41%,优于单独使用任一

3. **时间维度**:
   - 2018-2019 商誉暴雷 + 2022-2025 经济下行 = 风险双峰
   - 持续审计系统应重点关注这两个时期的财务数据

4. **行业差异化**:
   - 影视、地产、环保、化工是高风险行业
   - 金融、消费、农业相对稳定
   - 建议按行业设置不同的风险阈值

## 7. 局限性与展望

### 7.1 当前局限

1. **数据局限**:
   - 仅用 7 个财务比率,未纳入审计意见类型、内部控制评价、关联交易、担保等关键审计线索
   - 财务数据仅取违规则前 1 年,无法捕捉多年累积信号
   - 行业覆盖率 84.3%(退市/早期上市公司部分缺失)

2. **标签局限**:
   - LLM 预标注约 5-10% 误差,虽经 Agent 校验仍有噪声
   - `third_party_flag` 标签稀疏(321/8,724),模型难以学习
   - 负样本(ann_fin_flag=0)有限,不平衡比 1:0.35

3. **规则局限**:
   - 6 条规则严重度人为设定(基于经验),未做权重优化
   - 阈值固定,未按行业/规模差异化
   - R6(ROE 异常)在数据中 0 触发,可能阈值过严

4. **系统局限**:
   - 当前为离线分析,未实现实时数据接入
   - 异常检测参数(contamination=0.05)固定,未自适应
   - 仪表盘未做权限控制,生产环境需补充

### 7.2 未来展望

1. **数据扩展**:
   - 接入 tushare 审计意见、关联交易、担保信息
   - 引入文本特征:管理层讨论与分析(MD&A)语调
   - 多年度时序建模(LSTM/Transformer)捕捉累积风险

2. **模型升级**:
   - 引入 XGBoost / LightGBM 对比 RF
   - 尝试 SMOTE 过采样 + 类别权重的组合策略
   - 引入 SHAP 提供单家公司级解释

3. **实时化**:
   - 接入实时行情数据(日/小时级)
   - 触发规则后自动生成工单
   - 与审计系统集成,形成闭环

4. **跨行业应用**:
   - 模板迁移到采购付款审计、专项资金审计等其他场景
   - 复用本系统的"ML + 规则 + 异常检测 + 仪表盘"四件套架构
